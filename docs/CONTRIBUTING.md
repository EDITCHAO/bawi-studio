# Guide de Contribution - Bawi Studio

## 🤝 Comment contribuer

Merci de votre intérêt pour contribuer à Bawi Studio!

## 📋 Avant de commencer

1. Lire le `README.md`
2. Lire le `BEST_PRACTICES.md`
3. Lire le `PROJECT_STRUCTURE.md`
4. Configurer l'environnement de développement

## 🚀 Processus de contribution

### 1. Fork le projet

Aller sur GitHub et cliquer sur "Fork"

### 2. Cloner le fork

```bash
git clone https://github.com/votre_username/bawi-studio.git
cd bawi-studio
```

### 3. Créer une branche

```bash
git checkout -b feature/ma-fonctionnalite
```

### 4. Faire les changements

Modifier les fichiers et tester localement

### 5. Commiter les changements

```bash
git add .
git commit -m "Description claire du changement"
```

### 6. Pousser les changements

```bash
git push origin feature/ma-fonctionnalite
```

### 7. Créer une Pull Request

Aller sur GitHub et créer une Pull Request

## 📝 Conventions de commit

```
<type>(<scope>): <subject>

<body>

<footer>
```

### Types

- `feat`: Nouvelle fonctionnalité
- `fix`: Correction de bug
- `docs`: Documentation
- `style`: Formatage du code
- `refactor`: Refactorisation du code
- `perf`: Amélioration des performances
- `test`: Ajout de tests
- `chore`: Tâches de maintenance

## 🧪 Tests

### Avant de commiter

```bash
# Tester localement
npm run dev
cd backend && npm run dev

# Vérifier les logs
# Pas d'erreurs dans la console
```

## 📚 Documentation

### Mettre à jour la documentation

1. Modifier les fichiers `.md` correspondants
2. Vérifier que la documentation est claire
3. Ajouter des exemples si nécessaire
4. Commiter les changements

## 🐛 Signaler un bug

1. Aller sur GitHub > Issues
2. Cliquer sur "New Issue"
3. Remplir le titre et la description
4. Ajouter des logs et screenshots
5. Cliquer sur "Submit new issue"

## 💡 Proposer une fonctionnalité

1. Aller sur GitHub > Issues
2. Cliquer sur "New Issue"
3. Remplir le titre et la description
4. Ajouter le label "enhancement"
5. Cliquer sur "Submit new issue"
